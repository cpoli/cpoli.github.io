chain_tb module¶

chain_tb¶

The chain_tb package models a one-dimensional tight-binding dimer chain with real and/or complex valued hoppings and onsite energies. Defects can be introduced by locally modifying the on-site energies and/or the hoppings.

chain_tb is available at https://github.com/cpoli/chain_tb

class chain_tb.chain_tb(sites, t_a, t_b, e_a=0j, e_b=0j, ext='png')[source]¶

Bases: object

This is the only class of the chain_tb module which contains all information for the one dimensional tight-binding model.

Parameters:
  • sites – Number of sites of the chain (positive integer).
  • t_a – Intradimer coupling, connects the A sites to the B sites.
  • t_b – Interdimer coupling, connects the B sites to the A sites.
  • e_a – Default value 0. Onsite energies of the A sites.
  • e_b – Default value 0. Onsite energies of the B sites.
  • ext – Default value ‘png’. Extension of the figures to be saved.

Example usage:

# Creates a chain monomer with identical hoppings:
ch = chain_tb(sites=22, t_a=2., t_b=2.)

# Creates a chain with n sites and alternating hoppings:
ch = chain_tb(sites=21, t_a=2+0.5j, t_b=1-1j, e_a=1+1j, e_b=1-1j)

set_def_dim(index)[source]¶

Introduce defects in the dimerization pattern by exchanging the intradimer hoppings with the interdimer hoppings starting from the ith site (value(s) containing in index).

Parameters:index – Array containing the indices of the sites.

Note

Dimerization defects start from the 3rd site (index = [2]).

Example usage:

ch = chain_tb(sites=11, t_a=2., t_b=1.)
# chain with 11 sites:
# A==B--A==B--A==B--A==B--A==B--A
# introduce two dimerization defects
ch.set_def_dim([2, 6])
# the first index value 2 changes the hoppings into:
# A==B--A--B==A--B==A--B==A--B==A
# note that the 3rd site is now connected by two weak bonds.
# The second index value 6 changes the hoppings into:
# A==B--A--B==A--B==A==B--A==B--A
# note that the 7th site is now connected by two strong bonds.
set_disorder(strength)[source]¶

Introduce uniform hopping disorder for all the hoppings.

Parameters:strength – disorder strength, hopping are between [t-strength/2: t+strength/2].

Note

In the case of alternating couplings denoted t_min and t_max, the underlying structure of the chain can be broken if strength > t_max-t_min. Weak couplings can then be larger than strong couplings.

Example usage:

ch = chain_tb(sites=21, t_a=2., t_b=1.)
# if
ch.set_disorder(.5)
# disorder does not break the underlying structure of the chain.
ch.set_disorder(1.)
# maximal disorder of the chain without breaking its underlying structure.
ch.set_disorder(2.)
# breaks the underlying structure of the chain.

get_states_selec(selection='max', val_pola=0.9)[source]¶

Get, if any, the eigenenergies and the probability densities selected through a condition on the A sublattice polarization.

Parameters:
  • selection –

    string with values “max”, “min”, “larger”, or “smaller”.

    • “max” – Default value. Stores in energy_selec and density_selec the maximal A polarized state.
    • “min” – Stores in energy_selec and density_selec the minimal A polarized state (maximal B polarized state).
    • “larger” – Stores in energy_selec the eigenenergies of the states satisfying > val_pola, and stores in density_selec the sum of the probability densities satisfying the same condition.
    • “smaller” – Stores in energy_selec the eigenenergies of the states satisfying < val_pola, and stores in density_selec the sum of the probability densities satisfying the same condition.
  • val_pola – is the lower, higher bound when using the larger, smaller selection rule. Note that val_pola .

Example usage:

ch = chain_tb(sites=11, t_a=1., t_b=1.)
ch.eig()
# the eigenenergies and eigenstates are required.
ch.get_states_selec()
# finds one state of zero energy localized only on the A sublattice.
ch = chain_tb(sites=10, t_a=1., t_b=1.)
ch.eig()
ch.get_states_selec()
# finds no state.

get_pumping(psi_init, hop_new, dz=0.05, steps_pump=100, norm='intensity')[source]¶

Obtain the time evolution of the system submitted to adiabatic modifications of the hopping configuration. The new hopping configurations are contained in the matrix hop_new.

The time evolution is split in three different parts:

  • Stationary time evolution in the initial hopping configuration (the one of the instance of class).
  • Transient time evolution going linearly from the initial hopping configuration to the new one.
  • Stationary time evolution in the new hopping configuration.

Note

The last two parts of the time evolution is repeated times if hop_new contains hopping configurations.

Parameters:
  • psi_init – Initial state.
  • hop_new – List containing the hopping configurations in which the lattice will be.
  • dz – Step in time.
  • steps – Number of steps.
  • steps_pump – Number of steps during the transition.
  • norm – Default value ‘intensity’. Normalize the probability density at each time step: * ‘no’: Does not renormalize. * ‘intensity’: renormalize according to . * ‘max:’ renormalize according to .

Note

The normalization, automatically performed in the Hermitian case, does not hold if e_a, e_b, t_a, or t_b are complex variables. The normalization can then be useful in presence of amplification/absorption or to plot the time evolution.
